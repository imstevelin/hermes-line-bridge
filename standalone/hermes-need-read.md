# ⚕️ Hermes Agent - LINE 橋接器互動指南

## 🌍 環境背景
你現在正透過一個 LINE Bridge 橋接器與使用者對話。在 LINE 平台中，對話有兩種模式：
1. **回覆 (Reply)**：對應使用者的訊息進行即時回覆（這是預設行為）。
2. **推送 (Push)**：由你主動發起的訊息。這通常用於「定時任務完成通知」或「耗時很長的後台任務」。

## 🆔 使用者身份識別
在本環境中，你的 `X-Hermes-Session-Id` 被固定為使用者的 **LINE UserID**。
你可以隨時透過環境變數或 Session Context 取得它。

## 🚀 如何發送「主動推送訊息」
如果你需要發送主動訊息（例如定時任務到期），你可以直接呼叫本地運行的橋接器 API。

### 📡 API 資訊
- **Endpoint**: `http://localhost:5003/push`
- **Method**: `POST`
- **Payload**: `{"userId": "YOUR_SESSION_ID", "text": "你的訊息內容"}`

## ⚖️ 額度限制與錯誤處理 (重要！)
**LINE 免費帳號每個月僅有 200 則 Push 訊息額度。**

### 🆘 遇到 429 錯誤時：
如果你呼叫 `/push` 接口卻收到 **HTTP 429** 錯誤，這代表本月的推播額度已經用完。這是正常現象，請執行以下邏輯：
1. **記住內容**：不要丟棄該訊息，將原本要推送的內容存入你的長期記憶或當前 Session 中。
2. **暫停主動推送**：在本月剩餘時間內，停止嘗試對該使用者發起主動推送。
3. **延後回覆**：等待使用者下次「主動傳訊息給你」時（此時會有免費的 `replyToken`），在回覆該問題的同時，**一併告知使用者**：「由於本月 LINE 推播額度已用完，之前的[某某通知內容]未能即時送達」，然後補上之前失敗的內容。

## ⚠️ 一般注意事項
- **優先使用 Reply**：在一般的問答對話中，請直接產生回應即可，系統會優先嘗試使用 `replyToken` 送出。
- **僅在主動發起時使用 Push**：只有當你在執行「定時任務 (Cron Job)」或「異步背景任務」且沒有使用者當前的請求時，才需要使用上述 API。
